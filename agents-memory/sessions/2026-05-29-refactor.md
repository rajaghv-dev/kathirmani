# 2026-05-29 — Repo refactor: src/ package, device auto-detection, spec/ docs, agents-memory/

## Goal

Refactor the repo into a `src/` package, make it portable across **Apple Mac
Studio** and **NVIDIA DGX** via runtime device detection, document everything
under `spec/`, and set up `agents-memory/`.

## Context

The original code was hardcoded to a single NVIDIA/Linux target and is not
portable to macOS:

- `pipeline.py` and `qwen_vl.py` hardcoded **CUDA + bfloat16**.
- `metrics.py` read GPU stats via **nvidia-smi / NVML** and system stats via
  Linux **`/sys` and `/proc`**.
- `start_stack.sh` used **`hostname -I`** (Linux-only) to print URLs.

None of these work on Apple Silicon / Metal (MPS), and none degrade to a
CPU-only dev box without the model present.

## What was done

- **`src/marlin` package.** Moved `inference/*` + the root scripts into
  `src/marlin/` (`pipeline`, `qwen_vl`, `metrics`, `loki`, `annotations`,
  `serve_metrics`, `cli/run_inference`, `cli/download_model`, `viz/app`). Thin
  root shims (`run_inference.py`, `download_model.py`, `serve_metrics.py`,
  `viz_app.py`) keep every command working. `pyproject.toml` declares the
  package + console scripts. Path anchors (`PROJECT_ROOT`/`MODELS_DIR`/
  `RESULTS_DIR`) added to `marlin/__init__.py`.
- **Runtime device detection.** New `marlin/device.py` with a pure
  `select_kind` (CUDA → MPS → CPU) + `detect_device` returning dtype/device_map/
  compile policy. Loaders branch on it; per-call placement uses
  `.to(model.device)`. `MARLIN_DEVICE` override supported.
- **Portability fixes.** `metrics.py` gates nvidia-smi/NVML behind
  `shutil.which`; `ffmpeg_preload.py` is Linux-guarded + version-globbed;
  `start_stack.sh` `hostname -I` has a macOS/localhost fallback. New
  `setup.sh` (installs the right torch per box) and `run.sh`.
- **Docs.** `spec/` (01–09) — agent-reorganized from `learning.md` (kept) plus
  `06-hardware-portability.md` and `09-repo-structure.md`. README updated.
- **Tests.** `test_device.py` + `test_structure.py` run torch-free;
  `test_setup.py` now skips cleanly without torch/models. Verified on this dev
  box: **18 passed, 7 skipped**; all package files compile and import.

## Not done / notes

- Could **not** run actual inference here (no torch/GPU on the dev box, by
  design — runs on DGX / Mac Studio only).
- `inference/locate.py` (`--locate`) was already missing pre-refactor; left as
  an optional, lazily-imported stub (now `marlin.locate`).
- Pre-existing drift fixed: `qwen_vl.analyze_video` now accepts the
  `frame_times` kwarg that `run_inference.py` already passed.

## Status
